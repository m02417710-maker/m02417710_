الخطوة 1: استنساخ المستودع
────────────────────────────
git clone https://github.com/yourusername/stock-analyzer.git
cd stock-analyzer

الخطوة 2: نسخ ملف البيئة
────────────────────────────
cp .env.example .env

الخطوة 3: تحرير متغيرات البيئة
────────────────────────────
nano .env  # أو استخدم محررك المفضل

الخطوة 4: التشغيل مع Docker (موصى به)
────────────────────────────
docker-compose up -d

أو التشغيل المحلي:
npm install
npm run dev

الخطوة 5: الوصول
────────────────────────────
الواجهة الأمامية: http://localhost:3000
API: http://localhost:5000/api
قاعدة البيانات: localhost:5432
pgAdmin: http://localhost:5050
Redis Commander: http://localhost:8081

⚠️ قبل الإنتاج (Before Production):

1. غيّر جميع المفاتيح والكلمات المرور الافتراضية:
   • JWT_SECRET
   • DATABASE_PASSWORD
   • REDIS_PASSWORD

2. قم بتفعيل HTTPS/TLS

3. أضف عناوين IP البيضاء (Whitelist)

4. فعّل المصادقة متعددة العوامل (MFA)

5. قم بإعداد المراقبة والسجلات

6. اختبر خطوات الاستعادة من النسخ الاحتياطية

7. انشر سياسة خصوصية واضحة

💡 نصائح للتطوير:

1. استخدم Git للتحكم في الإصدارات:
   git init
   git add .
   git commit -m "Initial commit"

2. قم بتشغيل الاختبارات بانتظام:
   npm test

3. استخدم ESLint و Prettier:
   npm run lint
   npm run format

4. راقب الأداء:
   npm run health-check

5. احتفظ بالتوثيق محدثاً:
   npm run docs

6. اختبر في بيئات متعددة:
   • البيئة المحلية
   • الاختبار (Testing)
   • الإنتاج (Production)

7. استخدم git hooks:
   npm install husky --save-dev
   npx husky install
