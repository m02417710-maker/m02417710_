## الإبلاغ عن الثغرات الأمنية

### ⚠️ **لا تنشر الثغرات علناً**

إذا اكتشفت ثغرة أمنية، يرجى **عدم فتح Issue علني** لأن هذا قد يعرض المستخدمين للخطر.

### 📧 كيفية الإبلاغ:

أرسل بريد إلكتروني بالتفاصيل التالية إلى: **security@egxpro.com**

```
الموضوع: Security Issue - [اسم الثغرة]

المحتوى:
1. وصف الثغرة بوضوح
2. خطوات إعادة الإنتاج
3. التأثير المحتمل
4. البيئة (OS, Python Version, etc.)
5. لقطات شاشة أو أدلة (إذا أمكن)
6. اقتراحات الإصلاح (إذا كانت لديك)
```

### ⏱️ **المدة الزمنية للرد:**

- **الاستجابة الأولية**: خلال 48 ساعة
- **التحديث المنتظم**: كل 5 أيام
- **الإصلاح**: حسب الخطورة (متروك للحالة)

### 🎁 **البرنامج الحافز (Bounty Program):**

سيتم الاعتراف بالمبلغين عن الثغرات والشكر لهم في الإصدارة القادمة إذا رغبوا في ذلك.

   - اختبارات الأمان المنتظمة
   - فحص الكود الثابت (Static Analysis)

## الممارسات غير الآمنة (تجنبها!)

### ❌ لا تفعل:

```python
# لا تضع كلمات المرور في الكود
password = "admin123"
api_key = "sk_live_xxxxxxxx"

# لا تسجل البيانات الحساسة
logger.info(f"User password: {password}")

# لا تستخدم pickle للبيانات غير الموثوقة
import pickle
data = pickle.loads(untrusted_data)

# لا تستخدم eval
result = eval(user_input)

# لا تنسى السماح بـ CORS الكامل
CORS_ORIGINS = "*"
```

### ✅ افعل بدلاً من ذلك:

```python
# استخدم متغيرات البيئة
import os
password = os.environ.get("DB_PASSWORD")
api_key = os.environ.get("API_KEY")

# استخدم الصياغة الآمنة
logger.info("User authentication attempted")

# استخدم json.loads
import json
data = json.loads(user_input)

# استخدم ints و dicts والقوائم الآمنة
result = safe_eval(user_input)

# حدد CORS بحذر
CORS_ORIGINS = ["https://example.com"]
```
